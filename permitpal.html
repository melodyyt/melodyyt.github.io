PermitPal is an iOS app that is designed to scan a license plate to verify if that vehicle has a valid permit by checking against a database.

The objective of PermitPal is to replace the manual verification process of each car by introducing a digital and non-physical permit system.

Highlights:

Application is developed with SwiftUI.

The app's scanner functionality utilizes Apple's Vision framework.

The app pulls from Google Sheets using Google API to read from "database" populated by a Google Forms.

Used Agile scrum methodology to effectively manage and complete 10 week long project.

Here is a quick breakdown of the implementation

Step 1:

Implement scanner that can accurately read license plates.

Step 2:

Link iOS application to Google Forms and create comparison functionality. If inputed value from app is found in the Google Sheets return true, else false.

Step 3:

Put it together - send the output of the scanner into comparison function.

Step 1

Using Apple VisionKit's existing class that enables users to scan a physical document, parameters were adjusted so that the scanner only read text that were at least 80% height of the "document" or in this case, the license plate. Text such as the month, state, or registration sticker would be neglected.

Step 2

Upon pressing the button "press," the app will indicate whether the test plate "1234567" exists in the database, or Google Sheets. Initially, the entry "1234567" exists in the database, so the app returns true. When the entry is deleted, the app returns false.

Step 3

The final app requires users to sign in. There is a help page to explain how to navigate and utilize the app. To simulate a person purchasing a permit, the Google form is filled out. When that car is scanned, it will show that it has a valid permit with the green text. When a car that has not purchased a permit is scanned, the plate will show up with red text.

Future Considerations

  • Scanner can be perfected to read the edges of the license plate faster and more accurately.
  • In terms of accessability, indicated "valid" or "invalid" with additional text, rather than only color.
